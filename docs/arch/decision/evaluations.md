# 후보 구조 평가 결과 (Candidate Evaluations)

본 문서는 식별된 후보 구조(CA)들이 품질 시나리오(QS)에 미치는 영향을 평가하고, 최적의 아키텍처 결정을 위한 근거를 제시합니다.

## 1. 품질 시나리오 영향도 매트릭스

| ID | 후보 구조 제목 | QS-001 (성능) | QS-003 (변경) | QS-005 (확장) | QS-007 (유연) | 평가 요약 |
| :--- | :--- | :---: | :---: | :---: | :---: | :--- |
| **CA-101** | 전략 기반 검색 엔진 | (+) | (+) | - | (+) | 동적 전략 선택에 최적화됨. |
| **CA-102** | 파이프라인 기반 흐름 제어 | (-) | - | - | (-) | 구조가 직관적이나 동적 분기 처리에 경직됨. |
| **CA-103** | 추상화된 저장소 프로비저너 | - | (+) | - | - | 이종 DB 자원 할당의 일관성 보장. |
| **CA-104** | 후처리 인터셉터 체인 | - | (+) | - | (+) | 필터 추가/삭제 및 순서 변경이 용이함. |
| **CA-201** | 비동기 병렬 검색 흐름 | (++) | - | - | - | 응답 속도 체감 개선 효과가 가장 큼. |
| **CA-202** | 플러그형 엔진 어댑터 | - | (++) | - | - | 엔진 교체 시 도메인 로직 수정 최소화. |
| **CA-203** | 메시지 큐 기반 워커 분산 | - | - | (++) | - | 대량 문서 처리 시 시스템 안정성 확보. |
| **CA-204** | 런타임 전략 파라미터 전파 | - | - | - | (++) | 재시작 없는 실험 환경 제공의 핵심. |
| **CA-106** | 네임스페이스 기반 인덱스 분할 | - | - | (++) | - | 데이터 증가 시에도 검색 성능 상수 유지. |

> **범례**: `(++)` 매우 긍정, `(+)` 긍정, `(0) or -` 영향 없음, `(-)` 부정, `(--)` 매우 부정

## 2. 충돌 후보 구조 비교 평가

### 상충 그룹: CA-101 vs CA-102 (검색 엔진 구조)

| 품질 시나리오 | CA-101 (Strategy) | CA-102 (Pipeline) | 비교 우위 |
| :--- | :---: | :---: | :--- |
| **QS-001 (응답성)** | (+) | (-) | **CA-101** (오버헤드 적음) |
| **QS-007 (동적 설정)** | (++) | (-) | **CA-101** (런타임 분기 유리) |
| **가독성** | 도메인 핵심 로직 위주 | 프로세스 흐름 위주 | **CA-102** (직관적 흐름) |

- **트레이드오프**: CA-102는 고정된 순차 처리에 유리하나, RAGaaS의 핵심인 'Playground 실험(QS-007)'을 위해서는 런타임에 검색 경로를 동적으로 결정할 수 있는 **CA-101**이 필수적입니다.

---

## 3. 독립 후보 구조 및 조합 평가

### CA-201 (병렬 검색) 및 CA-201A (Throttling)
- **부정적 영향 완화**: CA-201 채택 시 일시적인 자원 경합(CPU/Connection)이 발생할 수 있으나, **CA-201A(세마포어 기반 제한)**를 함께 적용하여 시스템 다운타임을 방지합니다.

### CA-203 (비동기 워커) 및 CA-106 (인덱스 분할)
- **확장성 시너지**: CA-203은 처리량(Throughput)을 해결하고, CA-106은 검색 지연(Latency)을 해결하여 백만 건 이상의 대규모 데이터 환경(QS-005)을 완벽히 지원합니다.

---

## 4. 부정적 영향 완화 전략

### QS-001: 하이브리드 검색 응답 시간 (CA-102 기각 사유)
- **영향도**: (-)
- **영향 내용**: 파이프라인 단계 간 데이터 직렬화 및 전달 과정에서 불필요한 레이턴시 발생 가능성.
- **완화 전략**: CA-101(전략 패턴)로 대체하여 도메인 객체를 메모리 내에서 직접 공유.

### 인프라 자원 경합 (CA-201 관련)
- **영향도**: (-)
- **영향 내용**: 병렬 처리 시 동시 커넥션 수 초과 리스크.
- **완화 전략**: **CA-201A** 보완 후보 구조를 채택하여 엔진별 최대 동시 작업 수를 제한.
