# QS-001-하이브리드-검색-응답-시간

## 개요

### Quality Scenario ID
QS-001

### 제목
하이브리드 검색 응답 시간

### 설명
AI 애플리케이션이 질문(Query)을 던졌을 때, 시스템이 벡터 DB와 키워드 검색 결과를 결합하고 리랭킹을 거쳐 최종 결과를 반환하는 데 걸리는 전체 시간을 측정합니다.

### 품질 속성
성능 (Performance)

## 환경

### 시스템 상태
- 시스템이 정상 가동 중 (Normal Operation)
- 검색 대상 지식 베이스가 활성화되어 있으며 이미 인덱싱이 완료된 상태

### 초기 조건
- 데이터베이스(Milvus)에 약 10만 건의 텍스트 청크가 생성되어 있음
- 리랭커 모델 및 임베딩 모델이 메모리에 로드되어 있는 상태 (Warm-up 완료)

### 부하 조건
- 동시 사용자 1명 (Single User baseline)
- 네트워크 지연(Latency)은 무시 가능한 수준의 로컬 또는 인트라넷 환경

### 관련 컴포넌트
- AppAPI (Boundary)
- RetrievalEngine (Control)
- VectorStore (Entity)
- 외부 서비스 (Embedding/Rerank)

## 동작
1. AI 애플리케이션이 검색 요청 API를 호출한다. (Query 문자열 포함)
2. RetrievalEngine이 쿼리 임베딩을 위해 외부 임베딩 서비스에 요청을 보낸다.
3. RetrievalEngine이 VectorStore에 대해 ANN 검색과 키워드 검색을 병렬로 수행한다.
4. 검색 결과를 결합(RRF)한 후, 상위 후보들을 리랭커 서비스로 전달한다.
5. 리랭킹된 결과를 점수 정규화하여 AI 애플리케이션에 최종 응답한다.

## 측정
- **측정 항목**: 하이브리드 검색 End-to-End 응답 시간 (ms)
- **측정 공식**:
```
T_response = T_final_response_sent - T_request_received
```
