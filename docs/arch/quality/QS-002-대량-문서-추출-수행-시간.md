# QS-002-대량-문서-추출-수행-시간

## 개요

### Quality Scenario ID
QS-002

### 제목
대량 문서 추출 수행 시간

### 설명
사용자가 대규모 비정형 문서(예: 100페이지 분량의 기술 문서)를 업로드하고 지식 추출을 요청했을 때, 시스템이 문서 파싱부터 지식 트리플 저장까지 완료하는 총 처리 시간을 측정합니다.

### 품질 속성
성능 (Performance)

## 환경

### 시스템 상태
- 시스템이 정상 가동 중
- 백그라운드에서 다른 문서 처리 작업이 없는 단독 처리 상태 (Isolated Processing)

### 초기 조건
- 100페이지 분량 (약 5만 단어 내외)의 PDF 문서가 업로드되어 'Pending' 상태로 대기 중

### 부하 조건
- 시스템 자원(CPU, Memory)은 표준 할당량 내에서 사용 가능
- 지식 추출 서비스(LLM 연동)의 API Rate Limit이 충분히 확보된 상태

### 관련 컴포넌트
- IngestionEngine (Control)
- FileStore (Entity)
- VectorStore (Entity)
- GraphStore (Entity)
- 외부 추출 서비스 (Triple Extraction)

## 동작
1. 시스템 관리자가 특정 대용량 문서에 대해 '지식 추출 시작'을 요청한다.
2. IngestionEngine이 문서를 로드하여 텍스트를 파싱하고 정해진 크기로 청킹을 수행한다.
3. 각 청크에 대해 임베딩 벡터를 생성하고 VectorDB에 저장한다.
4. 문서 전체 또는 섹션별로 외부 서비스를 통해 엔티티 및 관계(Triple)를 추출한다.
5. 추출된 트리플을 GraphDB에 저장하고 문서 상태를 'Completed'로 변경한다.

## 측정
- **측정 항목**: 대용량 문서 처리 완료 시간 (초)
- **측정 공식**:
```
T_ingestion = T_status_completed_at - T_start_request_at
```
