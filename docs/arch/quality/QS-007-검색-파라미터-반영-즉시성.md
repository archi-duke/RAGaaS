# QS-007-검색-파라미터-반영-즉시성

## 개요

### Quality Scenario ID
QS-007

### 제목
검색 파라미터 반영 즉시성

### 설명
시스템 관리자가 플레이그라운드에서 검색 전략(ANN/Hybrid), 후처리 옵션(Rerank On/Off), 유사도 임계값(Threshold) 등을 변경했을 때, 서버 수정이나 재시작 없이 즉시 다음 검색 결과에 반영되는 유연성을 측정합니다.

### 품질 속성
사용성 (Usability) / 유연성 (Flexibility)

## 환경

### 시스템 상태
- 플레이그라운드 인터랙티브 모드 (Playground Session)
- 검색 엔진이 실시간 질의 처리 대기 중

### 초기 조건
- 검색 파라미터가 기본값(Default)으로 설정되어 검색 결과가 표시되고 있는 상태
- 변경할 파라미터가 시스템 설정(Config)이 아닌 요청 단위(Per-Request) 파라미터로 설계되어 있음

### 관련 컴포넌트
- AdminUI (Boundary)
- RetrievalEngine (Control)

## 동작
1. 시스템 관리자가 UI상에서 '리랭커 사용' 체크박스를 해제하고 '유사도 임계값'을 0.7에서 0.8로 상향 조정한다.
2. '검색' 버튼을 클릭하여 변경된 파라미터를 API 요청 바디(Body)에 담아 전송한다.
3. RetrievalEngine이 수신된 요청 내 파라미터를 즉시 해석하여 검색 파이프라인의 분기를 결정한다.
4. 변경된 로직에 따라 필터링된 검색 결과가 사용자 화면에 표시된다.

## 측정
- **측정 항목**: 설정 변경부터 결과 반영까지의 소요 시간 및 수동 작업 유무
- **측정 방법**:
  - 파라미터 수정 후 '검색' 실행 시, 변경된 파라미터가 결과셋(Result set)의 속성(예: 결과 건수 변화, 리랭킹 여부 로그 확인)에 즉각적으로 반영되는지 관찰
  - 설정 변경 시 서버 재시작 또느 설정 파일 수동 편집이 필요한지 확인 (Zero Manual Task)

## 관련 문서

- UC-203-검색 파이프라인 실험
- RetrievalEngine
