# QS-004-신규-임베딩-모델-추가-비용

## 개요

### Quality Scenario ID
QS-004

### 제목
신규 임베딩 모델 추가 비용

### 설명
최신 LLM 기술 트렌드에 따라 새로운 고성능 임베딩 모델(예: OpenAI v3, BGE-M3 등)을 시스템에 추가하거나 교체할 때 발생하는 공수와 시스템 영향도를 측정합니다.

### 품질 속성
변경 용이성 (Modifiability)

## 환경

### 시스템 상태
- 운영 중 또는 개발 환경
- 모델 연동을 위한 외부 서비스 인터페이스(Model Adapter)가 존재함

### 초기 조건
- 기존에는 HuggingFace 로컬 모델(예: sentence-transformers)을 사용 중
- 신규 모델은 API 방식(외부 유료 API) 또는 다른 프레임워크 기반의 모델임

### 관련 컴포넌트
- RetrievalEngine (Control)
- IngestionEngine (Control)
- ModelAdapterInterface (Boundary)

## 동작
1. 개발자가 신규 모델의 API 규격 또는 라이브러리 연동 로직을 포함하는 새로운 Adapter를 구현한다.
2. 시스템 설정 파일(Configuration)의 임베딩 모델 섹션에 새로운 모델 식별자와 파라미터를 등록한다.
3. 시스템을 재시작(또는 동적 로드)하여 신규 모델이 검색 및 수집 파이프라인에 적용되도록 한다.

## 측정
- **측정 항목**: 모델 추가 및 적용 비용
- **측정 방법**:
  - 신규 모델 도입을 위한 신규 클래스 구현을 제외한, 기존 핵심 로직(`Control`) 코드 수정 라인 수 (LOC)
  - 모델 교체 시 기존 메타데이터 구조(벡터 차원 수 등) 변경 필요 여부 분석에 소요되는 시간
  - 신규 모델 추가 후 전체 시스템 빌드 및 배포에 소요되는 시간
