# QS-005-백만-건-청크-규모-검색-성능-유지

## 개요

### Quality Scenario ID
QS-005

### 제목
백만 건 청크 규모 검색 성능 유지

### 설명
지식 베이스 내의 텍스트 청크 데이터가 백만 건(1M) 이상으로 대규모화되었을 때, 하이브리드 검색의 응답 속도와 유사 검색의 정확도(Top-K 관련성)가 일정 수준 이하로 급격히 저하되지 않는지 측정합니다.

### 품질 속성
확장성 (Scalability)

## 환경

### 시스템 상태
- 대량의 데이터가 로드된 고부하 운영 상태 (High Load Operation)
- 벡터 DB(Milvus)의 인덱스(HNSW 등)가 최적화되어 있는 상태

### 초기 조건
- 지식 베이스 내에 1,000,000개 이상의 유니크한 텍스트 청크 및 벡터 데이터가 인덱싱되어 있음
- 시스템 메모리(RAM)는 벡터 DB의 인덱스를 캐싱할 수 있는 임계치 근처에 도달함

### 부하 조건
- 지속적인 검색 요청(연속 100회 이상의 Query)이 발생하는 상황

### 관련 컴포넌트
- VectorStore (Entity)
- RetrievalEngine (Control)

## 동작
1. 외부 부하 테스트 도구가 무작위 키워드 및 의미론적 질의를 생성하여 검색 API를 호출한다.
2. RetrievalEngine이 대규모 인덱스를 가진 VectorStore에 대해 유사도 검색 및 필터링 검색을 수행한다.
3. 시스템은 검색 결과를 수집하여 점수를 계산하고 반환한다.

## 측정
- **측정 항목**: 데이터 규모에 따른 검색 성능 변화율
- **측정 방법**:
  - 데이터 10만 건 대비 100만 건 시점의 평균 응답 시간(Latency) 증가율 (%)
  - 대용량 데이터 환경에서의 검색 응답 시간의 표준 편차 (Jitter)
  - 검색 성능 유지를 위해 추가로 필요한 인프라 자원(Memory/CPU) 소모량 측정
