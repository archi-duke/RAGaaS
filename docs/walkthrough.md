# 그래프 데이터 정합성 & UI 수정 완료 보고서

이 문서는 삭제 시 데이터 정합성을 보장하고, 고아 데이터 문제를 해결하며 그래프 데이터 UI를 개선한 내용을 정리한 보고서입니다.

## 1. 백엔드: 트랜잭션 방식의 삭제 로직 구현 (고아 데이터 방지)
**목표**: 문서 삭제 후에도 그래프 노드나 청크가 남는 "고아 데이터(Orphaned Data)" 현상을 방지합니다.

### `cleanup_service.py` 변경 사항
"Leaf-to-Root" (외부 데이터 먼저 삭제 후 내부 기록 삭제) 방식의 엄격한 트랜잭션 로직을 구현했습니다.

```python
# 새로운 로직 의사코드 (Pseudo-code)
try:
    # 1. 외부 데이터 삭제 (Fuseki, Neo4j, Milvus)
    delete_fuseki_data(doc_id) # 실패 시 에러 발생 (멈춤)
    delete_neo4j_data(doc_id)  # 실패 시 에러 발생 (멈춤)
    delete_milvus_data(doc_id) # 검증 실패 시 에러 발생
    
    # 2. 삭제 후 검증 (Pre-Commit Verification)
    is_clean = verify_all_stores_are_empty(doc_id)
    if not is_clean:
        raise Error("잔여 데이터가 발견되었습니다! 삭제를 중단합니다.")
        
    # 3. 최종 커밋 (MongoDB 문서 기록 삭제)
    delete_mongodb_doc(doc_id)
    
except Error:
    # 단계 중 하나라도 실패하면 MongoDB 기록을 삭제하지 않고 남겨둡니다.
    # 이를 통해 "문서는 없는데 데이터만 떠도는" 좀비 상태를 방지합니다.
    # 사용자는 대시보드에서 에러를 확인하고 다시 삭제를 시도할 수 있습니다.
    log_error()
    abort()
```
- **효과**: 삭제가 중간에 실패하더라도 데이터의 일관성이 유지되며, 관리자가 문제를 인지하고 조치할 수 있게 됩니다.

## 2. 프론트엔드: 그래프 데이터 테이블 UI 개선
**목표**: 기존에 남아있던 고아 데이터를 우아하게 처리하고, 데이터 관계에 대한 오해를 줄입니다.

### `GraphDataTable.tsx` 변경 사항
| 문제점 | 수정 전 | 수정 후 |
| :--- | :--- | :--- |
| **청크 누락** | `⚠️ N/A` (경고 아이콘) | `-` (회색 처리, 무시 가능함 암시) |
| **문서 누락** | Raw UUID (예: `a1b2...`) | `ID... (del)` (삭제됨을 명시) |
| **중복 행 오해** | 같은 청크 ID가 반복되어 보임 | 툴팁 추가: *"하나의 텍스트 청크에서 여러 개의 트리플(지식)이 추출될 수 있습니다."* |

### 화면 예시 (Mockup)
**변경 전:**
```
| Document | Chunk      |
| a1b2c3d4 | ⚠️ N/A     |
```

**변경 후:**
```
| Document                | Chunk      |
| a1b2... (del)           | -          |
```
*청크 ID에 마우스를 올리면 1-to-N 관계에 대한 설명이 나옵니다.*
