# 작업 완료 보고서
(Last Updated: 2026-01-20)

## 1. [기능 추가] 부분 텍스트 추출 테스트 (Partial Selection Extract)
**목표**: 청크의 전체 내용이 길 경우, 사용자가 관심 있는 특정 부분만 드래그하여 추출 결과를 테스트할 수 있도록 합니다.

### 변경 사항
- **ChunkDetailModal.tsx**:
  - `useRef`와 `selectionchange` 이벤트를 사용하여 콘텐츠 영역 내의 텍스트 선택을 실시간으로 감지합니다.
  - "Extract" 버튼이 선택 상태에 따라 **"Extract"** 또는 **"Extract (Selection)"**으로 자동 변경됩니다.
  - 추출 요청 시, 선택된 텍스트가 있다면 해당 부분만 서버의 `extract` API로 전송됩니다.

### 사용 방법
1. 그래프 데이터 탭(또는 문서 탭)에서 **"Chunk"**를 클릭하여 상세 모달을 엽니다.
2. 우측 상단의 **"Extract Test"** 버튼을 눌러 설정 패널을 엽니다.
3. 콘텐츠 영역에서 원하는 문단을 **드래그하여 선택**합니다.
4. "Extract" 버튼이 **"Extract (Selection)"**으로 바뀌었는지 확인하고 클릭합니다.
5. 선택한 텍스트에 대해서만 추출된 트리플 결과를 확인합니다.

## 2. [버그 수정] 데이터 정합성 보장 및 UI 개선 (이전 작업)
**목표**: 문서 삭제 후에도 남는 "고아 데이터" 문제를 해결하고, 기존 데이터의 표시 오류를 개선했습니다.

### 백엔드 (트랜잭션 삭제)
- **삭제 순서 변경**: `Graph DB 삭제 → 검증 → MongoDB 삭제`.
- **안전 장치**: Graph DB 삭제가 실패하면 문서 기록을 지우지 않아 데이터 불일치를 방지합니다.

### 프론트엔드 (Graph UI)
- **누락 데이터 처리**: 청크 정보가 없는 경우 `⚠️ N/A` 대신 `-` (회색)으로 표시하여 사용자의 불안감을 줄였습니다.
- **삭제 흔적 표시**: 원본 문서가 삭제된 경우 ID 옆에 `(del)` 라벨을 표시합니다.
- **툴팁 추가**: 청크 ID에 마우스를 올리면 "하나의 청크에서 여러 트리플이 생성됨"을 설명하는 툴팁이 나옵니다.
