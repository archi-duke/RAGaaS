# Processing Pipeline Design Principles

## 청크 가공 vs ER 단계 책임 분리

문서 인제스트 파이프라인에서 전처리/후처리 작업의 책임을 명확히 구분합니다.

| 작업 | 담당 단계 | 시점 | 비고 |
|------|-----------|------|------|
| 번호/불릿/레이아웃 제거 | ✅ **청크 가공** | Chunking 전 | `text_cleaner.py` |
| 대소문자/공백 정리 | ✅ **청크 가공** | Chunking 전 | `text_cleaner.py` |
| 동의어/별칭 통합 | ❌ ER 단계 | Graph 추출 후 | 향후 구현 |
| 동일 엔티티 판단/병합 | ❌ ER 단계 | Graph 추출 후 | 향후 구현 |
| 추론 관계 생성 | ❌ Graph 추출 이후 | Graph 추출 후 | 향후 구현 |

## 원칙

1. **청크 가공**은 LLM/NLP 처리 **전에** 텍스트 노이즈를 제거하는 역할만 수행합니다.
2. **ER(Entity Resolution)** 단계는 그래프 추출 **후에** 엔티티 간 관계를 분석하고 병합합니다.
3. 두 단계의 책임을 혼합하지 않습니다.

> ⚠️ 이 원칙은 향후 기능 확장 시에도 준수해야 합니다.
